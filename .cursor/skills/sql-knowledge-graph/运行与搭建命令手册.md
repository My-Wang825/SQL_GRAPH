# SQL 知识图谱 — 运行与搭建命令手册

适用场景：GitHub 上的 [SQL_GRAPH](https://github.com/My-Wang825/SQL_GRAPH) 等仓库**仅有** `.cursor/skills/sql-knowledge-graph/` 下的 Skill 文档（及根目录 `README.md`、`.gitignore`、`LICENSE`），**没有** `backend/`、`frontend/`、`data/`、`run_server.py` 等实现代码时，按本手册把**参考实现**拷入当前仓库，即可在本机启动并完成**表级血缘可视化**。

Skill 正文中的「在新仓库落地的操作顺序」见 `SKILL.md`；本手册给出 **Windows PowerShell** 下可逐段执行的命令与校验步骤。

---

## 一、你需要事先准备的两样东西

| 项目 | 说明 |
|------|------|
| **Python** | 3.8 或以上，且 `python` 在终端可用（可用 `py -3` 代替下文中的 `python`）。 |
| **参考实现根目录** | 已包含完整代码的目录（本团队常用路径示例：`F:\knowledge-graph\SQL-graph`）。下文用环境变量 `$Ref` 表示，**请改为你机器上的真实路径**。 |

若你还没有参考实现目录，需要向团队索取完整工程压缩包，或从团队内部的完整 Git 仓库克隆后再继续第二节。

---

## 二、确认当前「仅 Skill」仓库根目录

在 PowerShell 中进入**仅含 Skill 的项目根**（根下应能看到 `.cursor\skills\sql-knowledge-graph\`，且尚无 `backend` 文件夹）：

```powershell
cd C:\你的路径\SQL_GRAPH
# 可选：确认当前没有后端
Test-Path .\backend
# 若为 False，说明需要执行下文复制步骤
```

下文用 `$Dest` 表示该目录，可直接用 **当前目录**：

```powershell
$Dest = (Get-Location).Path
```

---

## 三、从参考实现复制代码与数据（核心步骤）

### 3.1 设置参考实现路径

```powershell
$Ref = "F:\knowledge-graph\SQL-graph"
```

将 `$Ref` 改为你的完整项目路径。

### 3.2 复制目录与入口文件

以下命令**不会**覆盖你已存在的 `.cursor`（Skill 仍在原位置）；会向 `$Dest` 写入运行所需的实现文件。

```powershell
# 后端、前端、数仓 SQL 样例数据
Copy-Item -Path (Join-Path $Ref "backend") -Destination (Join-Path $Dest "backend") -Recurse -Force
Copy-Item -Path (Join-Path $Ref "frontend") -Destination (Join-Path $Dest "frontend") -Recurse -Force
Copy-Item -Path (Join-Path $Ref "data") -Destination (Join-Path $Dest "data") -Recurse -Force

# 启动脚本与依赖清单
Copy-Item -Path (Join-Path $Ref "run_server.py") -Destination $Dest -Force
Copy-Item -Path (Join-Path $Ref "requirements.txt") -Destination $Dest -Force
```

### 3.3（可选）环境变量模板与文档

```powershell
if (Test-Path (Join-Path $Ref ".env.example")) {
  Copy-Item -Path (Join-Path $Ref ".env.example") -Destination $Dest -Force
}
# 团队开发说明（若参考实现中有）
foreach ($f in @("开发说明文档.md", "README.md")) {
  $src = Join-Path $Ref $f
  if (Test-Path $src) {
    Copy-Item -Path $src -Destination (Join-Path $Dest ($f + ".from-ref")) -Force
  }
}
```

说明：根目录 `README.md` 可能与 GitHub 上已有说明冲突，故复制为 `README.md.from-ref`，你可自行对比合并。若确定要用参考实现的说明覆盖当前根 `README.md`，可改为直接覆盖（请先自行备份）。

### 3.4（可选）指标血缘合并文件

若需与参考实现一致，在 **PRD_AL** 图谱中合并指标平台治理 JSON，请一并复制：

```powershell
$exp = Join-Path $Dest "exports\metrics_lineage_test"
New-Item -ItemType Directory -Force -Path $exp | Out-Null
Copy-Item -Path (Join-Path $Ref "exports\metrics_lineage_test\lineage_for_governance_merge.json") -Destination $exp -Force -ErrorAction SilentlyContinue
```

不配该文件时，主图谱仍可运行，只是电解铝侧缺少指标合并边（见 `SKILL.md` / `reference.md`）。

### 3.5（可选）离线脚本

```powershell
if (Test-Path (Join-Path $Ref "scripts")) {
  Copy-Item -Path (Join-Path $Ref "scripts") -Destination (Join-Path $Dest "scripts") -Recurse -Force
}
```

## 四、复制结果自检（应存在的路径）

在 `$Dest` 下执行：

```powershell
@("backend\main.py", "backend\config.py", "frontend\index.html", "frontend\js\app.js", "data\PRD_AL", "run_server.py", "requirements.txt") | ForEach-Object {
  "$_ -> " + (Test-Path (Join-Path $Dest $_))
}
```

每一项都应为 `True`。若有 `False`，回到第三节检查 `$Ref` 路径与复制命令。

## 五、创建虚拟环境并安装依赖

**必须在项目根目录**（含 `backend`、`frontend`、`data` 的目录）执行：

```powershell
cd $Dest
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
pip install -r requirements.txt
```

若执行策略禁止激活脚本，可先运行：

```powershell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```

或使用：

```powershell
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
```

## 六、环境变量（可选）

智能问答、指标平台摘要等需要密钥；**仅浏览图谱**时通常可不配 `.env`。

```powershell
if (Test-Path .\.env.example) {
  if (-not (Test-Path .\.env)) { Copy-Item .\.env.example .\.env }
  notepad .\.env
}
```

在编辑器中填写你自己的 API 与内网地址；**勿将含真实密钥的 `.env` 提交到 Git**（仓库根 `.gitignore` 应包含 `.env`）。

---

## 七、启动服务

仍在项目根：

```powershell
.\.venv\Scripts\Activate.ps1
python run_server.py
```

终端会打印类似 `[监听] 127.0.0.1:8000` 及本机访问地址（端口被占用时会自动换端口，**以终端显示为准**）。

开发热更新（可选）：

```powershell
python run_server.py --reload
```

## 八、浏览器验证

将 `<端口>` 换成上一步终端中的端口（常见为 `8000`）：

| 步骤 | URL | 期望结果 |
|------|-----|----------|
| 健康检查 | `http://127.0.0.1:<端口>/health` | JSON：`{"status":"ok"}` |
| 电解铝图谱页 | `http://127.0.0.1:<端口>/` | 力导向图加载 |
| 氧化铝图谱页 | `http://127.0.0.1:<端口>/ao.html` | 同上，数据为 PRD_AO |
| 接口抽查 | `http://127.0.0.1:<端口>/api/graph?sector=PRD_AL` | 返回含 `nodes`、`links`、`meta` 的 JSON |

若页面提示无法加载图谱，多为**未在项目根启动**或端口不一致。

## 九、与 uvicorn 等价的启动方式（可选）

```powershell
.\.venv\Scripts\python.exe -m uvicorn backend.main:app --host 127.0.0.1 --port 8000
```

同样需在**项目根**执行，以便 `backend/config.py` 中的 `BASE_DIR` 能解析到 `frontend` 与 `data`。

## 十、常见问题

| 现象 | 处理 |
|------|------|
| `ModuleNotFoundError: backend` 或找不到 `frontend` | 确认当前目录为项目根，且已复制 `backend`、`frontend`。 |
| WinError 10013 / 端口占用 | 使用 `python run_server.py` 自动换端口，或改用 `--port 8080`；局域网访问需 `--host 0.0.0.0`（见参考实现根目录 `README.md.from-ref` 或团队文档）。 |
| 图谱为空 | 检查 `data\PRD_AL`（或 `PRD_AO`）下是否有 `.sql` / `.txt`；参考 `examples.md` 写一条最小 `INSERT…SELECT…FROM` 验证解析。 |
| 前端白屏、静态资源 404 | 确认 `main.py` 中静态目录挂载顺序未被人为改坏；以参考实现 `backend\main.py` 为准。 |

## 十一、提交到 Git 的建议

复制完成后，仓库将包含大量 `.py` 与 `.sql`，体积可能较大。建议：

- 确认 `.gitignore` 已忽略 `.venv/`、`.env`、`__pycache__/`。
- 大文件（如完整 `exports` 下原始 JSON）是否入库由团队策略决定；**Skill 不要求**必须提交 `lineage_queryAll_raw.json` 等大文件。
